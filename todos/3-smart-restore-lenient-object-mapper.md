- Priority: 3
- 목표: Lenient ObjectMapper를 추가해 Restore(스마트 복구) 포맷팅 로직에 통합
- 조사 메모:
  - Jackson 설정: ALLOW_SINGLE_QUOTES, ALLOW_UNQUOTED_FIELD_NAMES, ALLOW_COMMENTS/ALLOW_YAML_COMMENTS, ALLOW_TRAILING_COMMA(or ALLOW_COMMA_SEPARATOR) 활성화한 보수적 Mapper 생성.
  - 기존 서비스 구조: JsonFormatterService가 ObjectMapper 캐시/상태별 PrettyPrinter를 보유. Restore 모드 추가 시 캐싱/스레드세이프 고려.
  - 예외 처리: Lenient 파서 사용 후 정상화된 JSON을 표준 포맷으로 재생성해야 함(불량 입력 → 복구).
  - 설정 노출 여부: 사용자 설정에 lenient 모드 옵션 필요 여부 검토(기본 ON 오프라인 결정).
  - 성능: 대용량 JSON에서 파서 옵션 추가가 미미하지만, 캐시 공유로 재사용 필요.
