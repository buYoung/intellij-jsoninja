- Priority: 2
- 목표: JSONL(.jsonl), JSON5(.json5) 파일을 JSON 에디터에서 지원
- 조사 메모:
  - 파일 타입 정의: `FileTypeManager` 등록 여부 확인. IntelliJ 기본 JSON 파일타입이 `.jsonl`, `.json5`를 포함하는지 확인 필요(일부 버전 미포함 가능).
  - 편집기 연계: 현재 `EditorTextField` 생성 시 `JsonFileType.INSTANCE` 직접 지정 여부 확인 후, 확장자 인식에 따라 올바른 하이라이트/포맷팅이 동작하는지 검증.
  - 포맷터/서비스 영향: 포맷팅/검증/디프/랜덤 생성 로직이 MIME 또는 파일타입별 분기하는지 확인 필요. 대부분 문자열 처리라 영향 없을 수 있음.
  - 테스트/샌드박스: `.jsonl`, `.json5` 샘플 파일 로딩/포맷팅/검증 실행해보기.
- 다음 작업:
  - 코드 내 파일타입/확장자 하드코딩 검색(`rg -uu "\\.json"` 등)으로 제한 여부 확인.
  - 필요 시 커스텀 `JsonLikeFileType` 추가 또는 `FileTypeFactory`/`FileTypeEP`로 확장자 매핑.
  - 편집기 초기화 시 `.jsonl`/`.json5`도 JSON 하이라이팅/포맷팅 적용 확인.
  - QA: runIde로 샌드박스에서 열기/포맷/검증 동작 검증.
