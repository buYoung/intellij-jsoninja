# 도구 조합(워크플로) 벤치마크 결과 — C1~C6 × M1·M2·M3-W·M3-L, N=3

> 단일도구 벤치마크([`../codegraph-vs-serena/results.md`])가 "도구 하나의 실력"을 쟀다면, 이 문서는
> **도구를 파이프라인으로 엮었을 때 순이득이 있는가**를 같은 gold·blind·±1 채점으로 측정한다.
> 1차 실행(C1~C4): 2026-06-01, Workflow `wf_5e31d1d3-f29`(sonnet 49 agent = 워밍업 1 + 48셀), 1,414,506 토큰.
> 증분 실행(C5·C6): 2026-06-02, Workflow `wf_3a30b864-737`(sonnet 25 agent = 워밍업 1 + 24셀), 709,311 토큰.
> 채점은 결정적 `grader.py`(캐논 동결) + opus 종합/리뷰. **총 6조합 × 4케이스 × N=3 = 72 측정셀.**

## 1. 한 줄 결론

**조합의 순이득은 작고 케이스 한정적이며, 두 신규 끝점(C5·C6)이 그 한계를 더 또렷이 보여준다.** 단일도구(N=1)
대비 수치 향상은 여전히 **M2(동작 전파) recall 0.75 → 0.875**, **M3-L 정밀도 0.76 → 1.00** 둘뿐이고, 둘 다
serena/sg 없는 최단순 경로로 달성된다. 신규 두 조합은 공간의 양 끝을 못박는다.

- **C6(grep/find→read, 인덱스·심볼 전무)** 는 **6조합 중 비용 최저·속도 최고**(셀당 7.7 tool-use·55.8초)이고
  M1·M3-L 만점·M3-W 양호(0.73)지만, **M2 재현율이 치명적으로 불안정**(0.875 중앙값이나 ±0.331 — 전 연구 최대 분산,
  {0.375, 0.875, 1.0}).
- **C5(serena→read, 텍스트 인덱스 전무)** 는 **단일도구 serena의 약점을 그대로 재현**(M1 0.758·M2 0.75 — 타입
  미명시 DI 호출 누락)하면서 **비용은 6조합 중 최고**(캐시읽기 350K·tool-use 18.2). M3-W 에선 에이전트가
  `find_symbol` 로 기울어 **C1~C4 의 ctags 와 똑같은 선언부 오탐**(`TreeSitterWasmRuntime.kt:20-23`)을 물어 정밀도 0.667.

결론은 1차와 동일하되 더 강해진다: **여전히 C4(zoekt+ctags→read)가 비용·속도·품질의 최적 균형이다.** 심볼 도구를
얹으면(C1·C2·C5) 비싸지면서 M3-W 정밀도를 깎고 재현율 이득도 없으며, 텍스트만 쓰면(C6) 가장 싸지만 M2 폐포 추적이
운에 좌우된다.

## 4. 단일도구 baseline 대비 delta — 핵심 비교

기준선은 같은 데이터셋의 단일도구 결과([`../codegraph-vs-serena/results.md`] §3). C1~C4 는 1단계가 zoekt+ctags 이므로
**H(zoekt+ctags)** 가 1차 기준이고, **C5↔serena 단독(B)·C6↔bare/rg(E·D)** 가 직접 대응 끝점이다.

| 케이스 | 단일도구 최고 | 조합 최고(중앙값) | C5(serena+read) | C6(grep/find+read) | 해석 |
|---|---|---|---|---|---|
| **M1** recall | H/C/rg 1.00 | 1.00 (C1~C4·C6) | **0.758** | 1.00 | C5 만 천장 붕괴 — serena 단독(0.76) DI 맹점 재현. 텍스트(C6)는 만점 |
| **M2** recall | 0.75 | **0.875** (C2·C3·C4·C6) | 0.75 | 0.875 **(±0.331)** | 향상은 read/샘플링 효과(C4 로도 달성). C6 는 중앙값 동률이나 운에 좌우 |
| **M3-W** precision | serena 1.00 / zoekt 1.00 / H 0.67 | C4·C3 0.667 | 0.667 | **0.727** | 심볼 끼면(C1·C2·C5) 악화. C6(텍스트)가 신규 중 최고 |
| **M3-L** precision | serena 1.00 / H 0.76 | 1.00 (전 조합) | 1.00 | 1.00 | read/확인 단계가 enum 오탐 정리 |

**요지 넷**

1. **C5 는 serena 단독의 복제판이다(이득 없이 비용만).** M1 0.758(serena 단독 0.76)·M2 0.75(0.75)·M3-L 1.00(1.00)
   까지 단일도구 serena 와 거의 일치한다. read 를 더해도 **DI 수신자 호출(타입 미명시 `service<T>()`·필드 호출)을
   못 보는 구조적 맹점**은 그대로다 — 8곳을 일관되게 누락(M1 25/33 고정). 게다가 **셀당 캐시읽기 350K·tool-use 18.2 로
   6조합 중 가장 비싸다**(serena 다중홉이 컨텍스트를 매 턴 재독). 즉 "심볼 정밀도"를 위해 최대 비용을 치르고도
   재현율은 최하위다.

2. **M3-W 오탐의 진짜 원인은 ctags 가 아니라 "심볼 도구의 선언부/호출부 혼동"이다(C5 가 증명).** 1차 보고는 오탐 정체
   `TreeSitterWasmRuntime.kt:20-23`(WASM 함수 **선언부**, 정답은 호출부 61-64)을 **ctags 단계** 탓으로 봤다. 그러나
   **ctags 가 전혀 없는 C5 도 `find_symbol` 로 정확히 같은 20-23 선언부를 물어왔다**(r1·r3). 즉 이 오탐은 ctags 고유가
   아니라 **심볼 인덱스(ctags) 든 LSP 심볼검색(serena find_symbol) 이든 공통으로** 선언 심볼을 후보로 끌어들이는
   현상이다. 반대로 **텍스트 검색(plain zoekt·rg·serena search_for_pattern·C6 grep)은 호출 패턴을 직접 매칭해 이를
   피한다** — 그래서 C6(0.727)가 심볼 낀 C1·C2·C5(0.50~0.667)보다 정밀하다. (C5×M3-W r2 는 대신 WasmMemoryBridge
   함정 19·36 을 물어 0.727.)
   - 주의: C5 프롬프트는 Rust 측을 `search_for_pattern`(serena 내장 텍스트검색)으로 회수하라고 권했으나, 세 런 모두
     **에이전트가 `find_symbol` 로 기울었다**. serena 단독이 M3-W 1.00 을 받은 건 search_for_pattern 덕이었는데,
     조합 맥락에선 심볼 도구를 먼저 잡는 경향이 관찰된다. 따라서 C5 의 0.667 은 "serena 가 텍스트검색을 안 써서"이지
     "read 가 정밀도를 깎아서"가 아니다 — **에이전트의 도구선택 편향 자체가 결과**다.

3. **C6 는 가장 싸고 빠르지만 M2 가 동전던지기다.** C6 는 인덱스·심볼·LSP 없이 grep+find+read 만으로 M1·M3-L 만점,
   M3-W 0.727(신규 최고)을 낸다. 그러나 **M2(transitive 폐포)는 {0.375, 0.875, 1.00} 으로 ±0.331**(전 72셀 최대 분산).
   중앙값 0.875 가 C4(±0.07)와 같아 보여도 **재현성이 전혀 다르다** — 다중홉 역추적을 텍스트 grep 으로만 하면 사슬을
   끝까지 따라갈지가 run 마다 갈린다. M2 한 케이스에서 C6 는 "되면 8/8, 안 되면 3/8"이다.

4. **부수 이득(M3-L)은 전 조합 공통.** read/확인 단계가 enum 오탐을 잘라 H 의 0.76 을 1.00 으로 끌어올린다(C5·C6 포함).

> ⚠ **비교 주의**: 단일도구 baseline 은 **N=1**, 조합은 **N=3 중앙값**이다. 방향성 결론(C5=serena 복제·고비용,
> C6=최저비용·M2 불안정, 심볼 도구의 M3-W 선언부 오탐)은 N=3 분산까지 봐도 유효하다.

## 5. 케이스별 심층

### M1 — 시그니처 영향 (정답 33) ★ C5 가 유일하게 무너지는 곳
- C1~C4·C6: 1.00/1.00. 함정(동명 `JsoninjaPanelPresenter.formatJson`) 정확 배제.
- **C5: 재현율 0.758(25/33) 고정(±0)** — 단일도구 serena(0.76)와 동일. `find_symbol`+`find_referencing_symbols`
  로는 **타입을 명시하지 않고 DI 로 주입받은 필드를 통한 호출 8곳**을 일관 누락. read 를 더해도 회복 못 함.
  **C6(grep)은 같은 8곳을 텍스트로 다 잡아 1.00** — 이 케이스에서 텍스트가 심볼을 이긴다.

### M2 — 동작 전파 / transitive 폐포 (정답 8) ★ 변별·분산 케이스
- **C2·C3·C4·C6: 중앙값 0.875(7/8)** — 단일도구(0.75) 초과 지점. 단 C4 로도 달성(read/샘플링 효과).
- **C6: 0.875 중앙값이나 ±0.331(최대 분산)** — {0.375, 0.875, 1.00}. 텍스트 grep 만으로 다중홉을 추적하면 사슬
  말단(presenter·생성자)에서 run 마다 끊긴다. 안정성 최악.
- **C5: 중앙값 0.75(±0.22)** — serena 단독과 동일. r1 0.375(3개만 제출 후 종료) → serena 의 깊은 traversal 조기종료
  리스크 재현. C1(±0.26)과 같은 불안정 패턴.
- 공통 천장: `ConvertTypeDialogPresenter.<init>`(depth-5 DI 생성자)는 전 조합 누락.

### M3-W — 언어 간 경계 / ABI seam (정답 8 = Kotlin 4 + Rust 4, 함정 2)
- **재현율은 전 조합 1.00**. 문제는 **정밀도**.
- **심볼 도구는 선언부 오탐을 공유한다**: C1~C4(ctags)·C5(serena find_symbol) 모두 `TreeSitterWasmRuntime.kt:20-23`
  (선언부)를 호출부로 오인. ctags 유무와 무관 → §4-2 의 핵심 정정.
- **정밀도 순서**: C2(0.50) < C1(0.53) < C3·C4·C5(0.667) < **C6(0.727)** < (단일도구 serena·zoekt·rg 1.00).
  **텍스트만 쓰는 C6 가 조합 중 최고**, 심볼 낀 것들이 최저. serena 단독의 1.00 은 search_for_pattern(텍스트) 덕이지
  심볼검색 덕이 아님이 C5 로 재확인된다.

### M3-L — enum 참조 (정답 29)
- 전 조합 1.00/1.00(중앙값). C5·C6 도 read/확인으로 H 의 정밀도 약점(0.76)을 1.00 으로 보완. **변별 없음.**

## 7. 안정성 (N=3 표준편차)

- 대부분 셀 SD=0. **불안정 핫스폿은 M2 재현율** — **C6×M2 SD 0.331(전 72셀 최대)**, C1×M2 0.26, C5×M2 0.22, C2 0.14, C3 0.13.

  안정성을 원하면 C4, M2 폐포의 균형은 C3.

## 8. 권고

1. **기본값은 여전히 C4(zoekt+ctags→read).** 4 케이스 중 3개 최상위 동급 + M2 중앙값 동률이면서 **M2 가 안정적**(±0.07)이고
   비용·속도도 좋다. 신규 두 끝점을 본 뒤에도 결론 불변.
2. **비용·속도가 절대 우선이고 M2(폐포)가 무관한 작업이면 C6(grep/find→read).** 인덱스·심볼·LSP 없이 6조합 최저비용으로
   M1·M3-L 만점, M3-W 최고. **단 M2 는 쓰지 말 것**(±0.331, 되면 8/8·안 되면 3/8). "전수 역호출"이 필요하면 C4.
3. **C5(serena→read)는 비권장.** 단일도구 serena 를 그대로 복제해 **DI 호출 누락(M1 0.758·M2 0.75)** + **최고 비용**.
   심볼 정밀도가 정말 필요하면 단일 serena 를 쓰되, **M3-W 같은 언어 경계엔 find_symbol 이 선언부 오탐을 물므로
   search_for_pattern(텍스트) 을 강제**해야 한다.
4. **언어 경계(M3-W)엔 심볼 도구(ctags·serena find_symbol)를 얹지 말 것.** 선언부/호출부 혼동으로 정밀도가 깎인다.
   조합 중엔 텍스트만 쓰는 C6 가 최선이고, 근본적으로는 plain zoekt·rg·serena search_for_pattern 단일도구가 1.00 이다.
5. **C2(serena-first)는 비권장.** 최고 비용·최저 속도에 품질 이득 없고 M3-W 정밀도 최악.
